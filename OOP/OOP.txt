(유저시나리오) 사용자 아이디 입력 -> 식당 선택 -> 메뉴 선택 -> print(“0000원이 결제되었습니다.”)
입출력한다.
json 파일을 사용 혹은 json 형식의 하드코딩된 데이터를 임시사용.
총 식당수는 3개, 아침/점심/저녁은 특정 시간대를 체크하여 자동으로 적용되며. 해당 시간은 관리자가 추후에 쉽게 변경할 수 있어야 한다.
가게를 구분할때 사용되는 IF문에 직접적인 식당코드나 식당명을 사용하지 않는다.
if(식당코드==’식당코드_밥플러스’’)  -> 사용안됨 X. 다른방법. 즉 클래스의 특성을 이용할 것.
App내에 메뉴가 제공되는 가게가 있고 제공되지 않는 가게가 있다.
클래스간의 관계 설정은 반드시 한 번 이상은 사용하여야 한다.
다음의 클래스들은 필수로 존재하여야 한다. 나머지 클래스들은 필요에 의해 자유로이 추가한다.

Account (계정)
